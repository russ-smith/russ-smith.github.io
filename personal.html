Tess

After a while writing scripting code and shaders to run on top of existing game engines, I decided to build a simple game from the ground up using C++ and OpenGL.

Tess is the result of that experiment. It's a clone of the casual game 2048, played on a tesseract (a 4D cube).

Instead of a 4*4 tile game space, it's 2*2*2*2 with 4 possible movement directions from any point.

{% include tessimages.html %}

Features

Asset loading for shader files, BMFont fonts built with Hiero, and textures from PNG (I borrowed the decoder for this last one from LodePNG).

Persistent high score saved as hashed binary file

Uses instanced rendering with uniform buffers to draw the whole game in 5 draw calls

Meshify

This is an ongoing personal project based on a final-year project I made for uni.

Built using openFrameworks, it creates indexed 3D meshes from distance field functions, supplied as snippets of GLSL code.

All mesh extraction is done on GPU using compute shaders and 'histogram pyramids' to give interactive speeds on most recent graphics cards

Still very much an alpha version. The GUI needs a cleanup, and I should probably port the whole thing to something more friendly like Qt, but the core functionality works well.

{% include meshifyimages.html %}

Features

Two mesh creation algorithms - Marching Cubes and Dual Marching Cubes

Supports mesh resolutions from 64^3 to 512^3

Exports meshes to .obj files.
